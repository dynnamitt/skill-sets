# Map Shapes & Storage

Hexes can be stored in many layouts. Pick the shape from your gameplay
needs; pick the storage from the shape.

## Map shapes

### Hexagonal (radius `N`)

A roughly circular region — every hex within distance `N` of the center.
Generated by `range(center, N)`. Useful for symmetric maps without
preferred axes.

```
|q| ≤ N AND |r| ≤ N AND |s| ≤ N
```

Count: `3N² + 3N + 1`.

### Parallelogram (`Q × R`)

```
0 ≤ q < Q AND 0 ≤ r < R
```

Count: `Q · R`. Sheared rectangle in screen space; suits maps where you
don't mind the shear.

### Triangle (`N`)

```
q + r ≤ N AND q ≥ 0 AND r ≥ 0
```

Count: `(N+1)(N+2)/2`.

### Rectangle (`W × H`)

The screen-aligned rectangle is most natural in offset/doubled
coordinates. In axial, you get a sheared region — to make it screen-aligned
you have to compensate `q` based on `r`'s parity:

```
for r in 0..H:
    r_offset = floor(r / 2)
    for q in -r_offset..(W - r_offset):
        yield (q, r)
```

(Pointy-top with `oddr` offset; flat-top is the transpose.) This is what
`hexx::shapes::flat_rectangle` and equivalent honeycomb-grid traversers
do internally.

## Storage strategies

### `HashMap<Hex, T>` — universal

Works for any shape, including irregular and sparse maps. Slow for large
dense grids: each lookup is a hash + probe, ~10–20 ns versus ~1 ns for
an array index.

Use when:

- Map is irregular or sparse (e.g., dungeon rooms scattered across hex
  space)
- Map shape changes at runtime
- You need set operations (overlay, intersect)

### Flat array with `(q, r) → index` function — dense

For known shapes, derive an index from coordinates:

| Shape | Index formula |
|---|---|
| Parallelogram `Q × R` | `q + r * Q` |
| Rectangle `W × H` (oddr) | `(q + floor(r/2)) + r * W` |
| Hexagonal radius `N` | More involved; see hexx `HexagonalMap` |

15–100× faster than `HashMap`. Worth it for grids over ~100 hexes that
don't change shape.

`hexx` ships these as:

| Type | Shape |
|---|---|
| `HexagonalMap<T>` | Circular |
| `RombusMap<T>` | Parallelogram |
| `RectMap<T>` | Rectangle |

For TypeScript, write the indexer yourself:

```typescript
class DenseHexMap<T> {
  constructor(
    private cols: number,
    private rows: number,
    private values: T[]
  ) {}

  static parallelogram<T>(q: number, r: number, init: (q: number, r: number) => T) {
    const values = []
    for (let row = 0; row < r; row++)
      for (let col = 0; col < q; col++)
        values.push(init(col, row))
    return new DenseHexMap(q, r, values)
  }

  get(q: number, r: number): T {
    return this.values[q + r * this.cols]
  }
}
```

### 2D array — only with offset/doubled

```typescript
const map: Cell[][] = Array.from({ length: H }, () => new Array(W))
map[row][col] = cell
```

Only works if you store in offset or doubled. Convert at the boundary if
your algorithms run in axial.

## Wraparound maps

A hex map that wraps around (left edge connects to right edge, or
top-bottom, or both — like a torus or sphere) needs a way to "fold"
out-of-bounds coordinates back into the map.

### Mirror centers (Red Blob's approach)

Define `N` mirror centers — virtual map centers placed symmetrically
around the real map. Any hex outside the real map can be mapped back by
finding the nearest mirror center and translating.

For a hexagonal map of radius `R`, place 6 mirror centers at
`ring(origin, 2R + 1)` — one in each direction. To wrap a hex `h` back
into the map: find the mirror center `m` closest to `h`, and the wrapped
hex is `h - m`.

```
wrap(h, R, mirrors):
    if distance(h, ORIGIN) <= R:
        return h
    closest_mirror = min(mirrors, key=lambda m: distance(h, m))
    return h - closest_mirror
```

Works for movement, neighbors, range, FOV, A* — wrap every coordinate
returned from neighbor queries.

### `hexx::HexBounds`

Built-in. Wraps coordinates outside bounds back into a hexagonal region:

```rust
let bounds = HexBounds::new(Hex::ZERO, radius);
let wrapped = bounds.wrap(out_of_bounds_hex);
```

For TypeScript, implement the mirror centers approach yourself.

### Rectangular wraparound

Easier: `(col mod W, row mod H)` in offset coordinates. This is one case
where offset/doubled storage pays off.

## Picking storage

| Map | Storage |
|---|---|
| Catan-style fixed hexagonal (radius 2-3) | Doesn't matter; use HashMap, you'll never notice |
| Strategy game, 50×50 rectangular | 2D offset array, or dense `RectMap` if axial |
| Worldgen with chunks | Dense per-chunk, indexed by chunk coord |
| Sparse / irregular | `HashMap<Hex, T>` |
| Wraparound torus / sphere | Dense + wrap function |
| Multi-resolution (overview + detail) | hexx multi-res (`to_lower_res`) |
